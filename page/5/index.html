AWS BatchвҖӢ

кё°ліём ҒмңјлЎң Batch м»ҙн“ЁнҢ…мқ„ лӘ©м ҒмңјлЎң н•ҳлҠ” м„ңл№„мҠӨлЎң м»ҙн“ЁнҢ… нғҖмһ…мқ„ Fargate лҳҗлҠ” EC2 мӨ‘ н•ҳлӮҳлҘј м„ нғқн•ҳм—¬ лҸҷм ҒмңјлЎң н”„лЎңл№„м ҖлӢқн•ҳлҠ” м„ңл№„мҠӨмқҙлӢӨ.
мҙҲкё° м„Өм •мқҙ лІҲкұ°лЎӯкёҙ н•ҳм§Җл§Ң н•ңлІҲ кө¬м„ұн•ҳкі  лӮҳл©ҙ лі„лҸ„ кҙҖлҰ¬к°Җ н•„мҡ”м—Ҷкі  Batchмһ‘м—…мқҙ л°ңмғқн•ҳлҠ”кІғкіј м»Ён…Ңмқҙл„Ҳ мқҙлҜём§Җ ліҙкҙҖл№„мҡ© мқҙмҷём—җлҠ” м¶”к°Җ л№„мҡ©мқҙ л°ңмғқн•ҳм§Җ м•Ҡкё° л•Ңл¬ём—җ л№„мҡ©мёЎл©ҙм—җм„ң мһҘм җмқҙ мһҲлҠ” м„ңл№„мҠӨмқҙлӢӨ.
мң„м—җм„ң мқҙм•јкё°н•ң м •кё°м ҒмңјлЎң мғҲлІҪл§ҲлӢӨ нҒ¬лЎӨл§Ғ(Pull)л°©мӢқмңјлЎң лҚ°мқҙн„°лҘј ліөм ңн•ҙм•јн•ҳлҠ” Use-CaseлҘј кө¬нҳ„н•ҳлҠ”кІғмқҙ м ҒлӢ№н•ң мӣҢнҒ¬лЎңл“ң кө¬нҳ„л°©лІ•мқҙлқј мғқк°Ғн–Ҳкё° л•Ңл¬ём—җ AWS Batch м„ңл№„мҠӨлҘј мӮ¬мҡ©н•ҳкІҢ лҗҳм—ҲлӢӨ.

к°„лӢЁн•ң Batch demoлҠ” https://github.com/awslabs/aws-batch-helpers/м—җм„ң нҷ•мқёмқҙ к°ҖлҠҘн•ҳл©° CircleCIм—°лҸҷмқ„ мң„н•ҙ Forkнӣ„ м§„н–үн•ҳмҳҖлӢӨ.

Demo Repository : https://github.com/ddiiwoong/aws-batch-helpers/

лӘЁл“  PipelineмқҖ CircleCIкё°л°ҳмңјлЎң мһ‘м„ұн•ҳмҳҖлӢӨ. нҳ№мһҗлҠ” AWS Code Commit, Pipelineмқ„ мӮ¬мҡ©н•ҳм§Җ м•ҠлҠҗлғҗкі  л¬ёмқҳн•ҳмӢңлҠ” л¶„л“ӨлҸ„ мһҲлҠ”лҚ° лӢӨмқҢкіј к°ҷмқҖ мқҙмң лЎң CircleCIлҘј м„ нғқн•ҳмҳҖлӢӨ.

  • Fully Managed (Serverless)
  • AWS мў…мҶҚм„ұ мөңмҶҢнҷ”
  • Git, Registry, CDмҳҒм—ӯмқҳ нҷ•мһҘм„ұ кі л Ө

  • мүҪкі  лӢЁмҲңн•ҳкІҢ л№Ңл“ң нҷҳкІҪкө¬м„ұ
  • мҶҢк·ңлӘЁ н”„лЎңм қнҠё л№ лҘҙкІҢ мӢңмһ‘ к°ҖлҠҘ

TerraformвҖӢ

https://www.terraform.io/docs/providers/aws/index.html

м„ м–ём Ғ мқён”„лқјмҠӨнҠёлҹӯмІҳ кҙҖлҰ¬ лҸ„кө¬лЎң л§Һмқҙ мӮ¬мҡ©н•ҳкі  мһҲлҠ” лҸ„кө¬мқҙл©° DocsмҷҖ лё”лЎңк·ё мһҗлЈҢк°Җ л§ҺмқҖ кҙҖкі„лЎң л”°лЎң м„ӨлӘ…н•ҳм§Җ м•ҠкІ лӢӨ.
мқҙлІҲ нҸ¬мҠӨнҠём—җм„ңлҠ” AWS BatchлҘј мғқм„ұн•ҳлҠ” мҳҒм—ӯмқ„ Terraformмқҙ лӢҙлӢ№н•ңлӢӨ.

CircleCI Config мһ‘м„ұвҖӢ

version: 2.1
orbs:
  aws-ecr: circleci/aws-ecr@6.1.0
  aws-ecs: circleci/aws-ecs@0.0.8
  aws-s3: circleci/aws-s3@1.0.11
  aws-cli: circleci/aws-cli@0.1.13
jobs:
  sh-s3-upload:
    docker:
      - image: 'circleci/python:2.7'
    steps:
      - checkout
      - aws-s3/copy:
          from: ./myjob.sh
          to: 's3://batch-ecr-test/myjob.sh'
  deploy-batch:
    executor: aws-cli/default
    steps:
      - checkout
      - aws-cli/install
      - run:
          name: Update AWS Batch Job
          command: |
            aws batch register-job-definition --job-definition-name fetch_and_run --type container --container-properties '{ "image": "823928750534.dkr.ecr.ap-northeast-2.amazonaws.com/fetch_and_run:v20190623", "vcpus": 1, "memory": 512}'
workflows:
  build-and-deploy:
    jobs:
      - aws-ecr/build-and-push-image:
          repo: $AWS_RESOURCE_NAME_PREFIX
          tag: v20190623
      - sh-s3-upload:
          name: sh-s3-upload
          requires:
            - aws-ecr/build-and-push-image
      - deploy-batch:
          name: deploy-batch
          requires:
            - sh-s3-upload

лӢЁкі„лі„ м„ӨлӘ…мқ„ мң„н•ҙ л¶Җл¶„м ҒмңјлЎң м„ӨлӘ…н•ҳлҸ„лЎқ н•ҳкІ лӢӨ.

  1. TerraformмңјлЎң Batch л°°нҸ¬

    resource "aws_batch_compute_environment" "default"{ 
      compute_environment_name = "env_fetch_and_run" 
      compute_resources { 
        instance_role = "arn:aws:iam::823928750534:instance-profile/ecsInstanceRole" 
        instance_type = [
          "optimal",
        ]
        desired_vcpus = 1 
        max_vcpus = 1 
        min_vcpus = 0 
        security_group_ids = [
          "sg-0632cf81b5c4dff17"

          "subnet-0c4f8135b536e8fab",
          "subnet-0a261950d894cf27e"

    Batch Computing нҷҳкІҪмқ„ кө¬м„ұн•ҳкІҢ лҗҳлҠ”лҚ° "aws_batch_compute_environment"лҘј мӮ¬мҡ©н•ңлӢӨ.

    • compute_environment_name : (н•„мҲҳ) м»ҙн“ЁнҢ… нҷҳкІҪ мқҙлҰ„
    • compute_resources.instance_role : (н•„мҲҳ) EC2м—җ м Ғмҡ©лҗҳлҠ” Role
    • compute_resources.instance_type : (н•„мҲҳ) мӮ¬мҡ© к°ҖлҠҘн•ң instance мң нҳ•
    • compute_resources.desired_vcpus : (мҳөм…ҳ) мӣҗн•ҳлҠ” CPUмҲҳ
    • compute_resources.max_vcpus : (н•„мҲҳ) мөңлҢҖ CPUмҲҳ
    • compute_resources.min_vcpus : (н•„мҲҳ) мөңмҶҢ CPUмҲҳ
    • compute_resources.security_group_ids : (н•„мҲҳ) EC2м—җ м Ғмҡ©лҗҳлҠ” Security Group
    • compute_resources.subnets : (н•„мҲҳ) Subnets лӘ©лЎқ
    • compute_resources.type : (н•„мҲҳ) EC2лҘј кё°л°ҳмңјлЎң мғқм„ұ, SPOT instance мӮ¬мҡ©к°ҖлҠҘ
    • service_role : (н•„мҲҳ) AWS Batchк°Җ лӢӨлҘё AWS м„ңл№„мҠӨлҘј нҳём¶ң н• мҲҳмһҲкІҢ н•ҙмЈјлҠ” IAM Role(ARN)
    • type : (н•„мҲҳ) MANAGEDлӮҳ UNMANAGEDлҘј м„ нғқн•  мҲҳ мһҲкі , MANAGEDмқҳ кІҪмҡ° compute_resourcesм—җ м„ёл¶Җ мӮ¬н•ӯмқ„ м„Өм •н•  мҲҳ мһҲлӢӨ.
    resource "aws_batch_job_definition" "default" {
      name = "fetch_and_run" 
      type = "container"

      container_properties = <<CONTAINER_PROPERTIES
    {
        "command": [],
        "image": "823928750534.dkr.ecr.ap-northeast-2.amazonaws.com/fetch_and_run:v20190623",
        "vcpus": 1,
        "memory": 512,
        "volumes": [],
        "environment": [],
        "mountPoints": [],
        "ulimits": [] 
    }
    CONTAINER_PROPERTIES
    }

    Job Definition м •мқҳ("aws_batch_job_definition")м—җм„ңлҠ” Resource spec л°Ҹ Command(Docker RUN), Container IMAGE л“ұмқ„ м„ м–ён•ҳкІҢ лҗңлӢӨ.

    • name : (н•„мҲҳ) Job Definition мқҙлҰ„
    • type : (н•„мҲҳ) Job Definition мң нҳ•, ECSлЎң мІҳлҰ¬н•ҳкё° мң„н•ҙ containerлЎң м„ нғқ
    • container_properties : (м„ нғқ) JSON нҳ•нғңлЎң мһ‘м„ұн•ҳлҠ” container мҶҚм„ұм •ліҙ (Image, Resources, Command, MountPoint л“ұ)

  2. .circleci/config.ymlм—җ CircleCI лІ„м „ лӘ…мӢң л°Ҹ кҙҖл Ё orb(ecr,ecs,s3,cli) м¶”к°Җ
    кё°ліём ҒмңјлЎң 2.1лЎң м„Өм •мқ„ н•ҙм•ј ecs, eks orb мӮ¬мҡ©мқҙ к°ҖлҠҘн•ҳлӢӨ.

      orbs:

        aws-ecs: circleci/aws-ecs@0.0.8

        aws-cli: circleci/aws-cli@0.1.13

  3. workflow кө¬м„ұ

    build-and-push-image -> sh-s3-upload -> deploy-batch мҲңмңјлЎң мҲңм°Ём ҒмңјлЎң pipelineкө¬м„ұмқ„ н•ҳмҳҖлӢӨ.

    workflows:
      build-and-deploy:
        jobs:
          - aws-ecr/build-and-push-image:
              repo: $AWS_RESOURCE_NAME_PREFIX # data-crawler-test
              tag: v20190623
              # create-repo: true
              # dockerfile: Dockerfile
          - sh-s3-upload:
              name: sh-s3-upload
              requires:
                - aws-ecr/build-and-push-image
          - deploy-batch:
              name: deploy-batch
              requires:
                - sh-s3-upload

    кё°ліём ҒмңјлЎң CircleCIм—җм„ңлҠ” orb jobмқ„ [orb name]/[predefined-job] нҳ•мӢқмңјлЎң м„ м–ён•ңлӢӨ.

    I. мІ«лІҲм§ё stepм—җм„ңлҠ” container imageлҘј buildн•ҳкі  pushн•ҳлҠ” Jobмқ„ мҲҳн–үн•ҳлҜҖлЎң aws-ecr/build-and-push-imageмқ„ м„ м–ён•ңлӢӨ.
    мһҗм„ён•ң лӮҙмҡ©мқҖ https://circleci.com/orbs/registry/orb/circleci/aws-ecrлҘј нҷ•мқён•ҳмһҗ.

    II. л‘җлІҲм§ё stepм—җм„ңлҠ” S3м—җ л°°м№ҳмҠӨнҒ¬лҰҪнҠёлҘј мҳ¬лҰ¬лҠ” Custom Jobмқ„ мҲҳн–үн•ңлӢӨ. JobмқҖ Stepмқҳ лӘЁмқҢмңјлЎң м„ н–үлҗҳм–ҙм•јн•  JobмқҖ мғҒмң„м—җ requires: м—җм„ң м„ м–ён•ҳл©ҙ лҗңлӢӨ.

    III. м„ёлІҲм§ё stepм—җм„ңлҠ” AWS Batch job definitionмқҳ container imageлҘј көҗмІҙ(revision ліҖкІҪ)н•ҳлҠ” Custom Jobмқ„ мҲҳн–үн•ңлӢӨ.

  4. custom job кө¬м„ұ Workflowм—җм„ң м„ м–ён•ң Custom Jobмқҳ мғҒм„ё definitionмқ„ кё°мһ¬н•ңлӢӨ.

    I. sh-s3-upload м—җм„ңлҠ” aws-s3/copy лҘј мӮ¬мҡ©н•ҳм—¬ мӣҗн•ҳлҠ” S3лІ„нӮ·м—җ мӮ¬мҡ©н•  мҠӨнҒ¬лҰҪнҠёлҘј м—…лЎңл“ң н•ңлӢӨ. checkoutм—җм„ңлҠ” нҳ„мһ¬ мғҒнғңмқҳ git repoлҘј cloneн•ҳкІҢ лҗңлӢӨ.

    II. deploy-batch м—җм„ңлҠ” aws-cli/install лҘј мӮ¬мҡ©н•ҳм—¬ кё°мЎҙм—җ мһ‘м„ұн•ң Batch Job definitionмқ„ awscliлЎң мӣҗн•ҳлҠ” мғҲлЎңмҡҙ imageлЎң м—…лҚ°мқҙнҠё н•ҳкІҢ лҗңлӢӨ.

CircleCI мӢӨн–үвҖӢ

л№Ңл“ң л°Ҹ л°°нҸ¬кіјм •мқ„ к°„лӢЁн•ҳкІҢ м„ӨлӘ…н•ҳл©ҙ мҪ”л“ңк°Җ м—…лҚ°мқҙнҠёлҗҳл©ҙ CircleCIлҠ” н•ҙлӢ№ м ҖмһҘмҶҢмқҳ .circleci/config.yml мқ„ кё°мӨҖмңјлЎң л№Ңл“ң л°Ҹ л°°нҸ¬лҘј мӢңмһ‘н•ҳкІҢ лҗңлӢӨ.

README.mdм—җ Build Status Badge лӢ¬кё°вҖӢ

https://circleci.com/docs/2.0/status-badges/

мң„ л§ҒнҒ¬мҷҖ м„Өм •мқ„ м°ёкі н•ҳм—¬ м•„лһҳ к·ёлҰјкіј к°ҷмқҙ Build Statud BadgeлҘј лӢ¬ мҲҳ мһҲлӢӨ.

м •лҰ¬вҖӢ

мқҙлІҲм—җлҠ” CircleCIлҘј к°Җм§Җкі  AWS Batch jobмқ„ кө¬м„ұн•ҳлҠ” лҚ°лӘЁлҘј м§„н–үн•ҳмҳҖлӢӨ.

м–јл§Ҳм „мқёк°Җм—җлҸ„ м–ҙнҳ•л¶Җнҳ•лӢҳк»ҳм„ңлҸ„ м–ёкёүн•ҳмӢ  м„ м–ём Ғмқё кө¬м„ұ кё°л°ҳм—җ мөңлҢҖн•ң serverless нҷҳкІҪм—җм„ңмқҳ л°°нҸ¬лҘј м¶”кө¬н•ҳкІҢ лҗҳлӢӨ ліҙлӢҲ м•„лһҳмҷҖ к°ҷмқҖ CI/CD Pipeline мӢңлӮҳлҰ¬мҳӨлҘј кө¬мғҒн•ҳкІҢ лҗҳм—ҲлӢӨ.

мғҲлЎңмҡҙ кё°лҠҘмқҳ лёҢлһңм№ҳ(New Feature)к°Җ Gitм—җ pushлҗҳл©ҙ л№Ңл“ңмҷҖ н…ҢмҠӨнҠёк°Җ нҠёлҰ¬кұ°лҗҳм–ҙ мһҗлҸҷмңјлЎң м§„н–үлҗҳкі  лҸҷмӢңм—җ к°ңл°ңмһҗк°Җ PRмқ„ мһ‘м„ұн•ҳл©ҙ лҸҷлЈҢ л°Ҹ лӢҙлӢ№ мғҒкёүмһҗм—җкІҢ мҪ”л“ң лҰ¬л·°лҘј л°ӣкІҢ лҗңлӢӨ.

лҰ¬л·°к°Җ нҶөкіјлҗҳкі  CircleCIм—җм„ң л№Ңл“ңк°Җ м„ұкіөн–ҲлӢӨл©ҙ мҠ№мқёлӢЁкі„лҘј нҶөн•ҙ MergeлҘј н•ҳкі  CircleCI к°Җ н•ң лІҲ лҚ” л№Ңл“ңлҘј мӢңмһ‘н•ҳкі  л°°нҸ¬к№Ңм§Җ мҲҳн–үн•ҳкІҢ лҗңлӢӨ.

ECSлӮҳ BatchлЎң л°°нҸ¬лҠ” CircleCIлҘј нҶөн•ҙ м§Ғм ‘ л°°нҸ¬лҘј н•ҳкі  EC2лӮҳ EKSлЎңмқҳ л°°нҸ¬лҠ” Terraform л°Ҹ ArgoCDлҘј нҶөн•ҙ л°°нҸ¬лҘј м§„н–үн•ҳлҠ” л°©мӢқмқҙлӢӨ.

лӢӨмқҢлІҲ нҸ¬мҠӨнҢ…м—җлҠ” мң„ к·ёлҰј кё°л°ҳмңјлЎң CircleCIмҷҖ ArgoCDлҘј нҷңмҡ©н•ҳм—¬ EKSкё°л°ҳ л°°нҸ¬кіјм •мқ„ м •лҰ¬н•  мҳҲм •мқҙлӢӨ.
